# 技术路线 v3 — 专业极简版（数字员工管理系统）

> **澄清**：v2 误读了"10 岁能用"，把它做成了儿童化卡通 UX。v3 重新定义：
>
> "**10 岁能用**" = **零门槛、极简、自解释**，**不是**卡通、表情包、儿童视觉。
>
> 视觉与交互对标：**Linear · Notion · Raycast · Vercel · Stripe · Attio · Superhuman · Cursor**
>
> 一句话：**像 Linear 一样克制、像 Notion 一样自由、像 Raycast 一样快、像 Stripe 一样可信、像 Cursor 一样有 AI**。

## v3 vs v2 — 修订项总览

| 项 | v2 错的 | v3 修正 |
|---|---|---|
| 视觉风格 | "10 岁孩子可用" 被误解为 Memoji 头像、emoji + 微动画、卡通 | 严肃产品风：**单色 + 灰阶 + 1 抹品牌色**；动画只用于状态变化与位移；零 emoji 装饰 |
| SOP 编辑 | 三色块（绿/蓝/黄），明显模仿 Scratch | **行内文字 SOP**（自然语言 + 图标 + 关键词高亮），形如 Notion 文档；高级模式才出节点 |
| 头像/角色 | "捏脸"+ "心情"+ "奖金" | **单色字母头像**（Linear 风格），可换为公司 Logo；移除心情/奖金 |
| 模式切换 | "儿童 / 家庭模式" | **删除**。一个产品只服务一种语气：专业、克制、克制、再克制 |
| 字体/色彩 | 大号、彩色、大量 emoji | **Inter + JetBrains Mono**；4 个灰阶 + 1 个品牌色 + 4 个状态色 |
| 微交互 | confetti / pulse 多 | 仅保留：状态点呼吸（idle）、loading shimmer、150ms ease-out 过渡 |
| 用户测试 | 邀请 10 岁儿童 | **邀请非技术管理者** + 设计师 + 不熟悉本产品的开发者三类，**不再用儿童** |
| OKR 指标 | "10 岁孩子 5 分钟雇佣员工" | "**非技术管理者 5 分钟内雇佣并看到首个真实结果**" |

> 删除的：儿童/家庭模式、心情系统、奖金系统、Memoji、Scratch 块、emoji 装饰、micro-confetti
>
> 保留的：公司画布、Captain 副驾、思考剧场、模板市场、撤销一切、可分享录像、行业方案包、SOP 自然语言改、人工审批

## Part A — 设计原则 v3（7 条，专业极简）

### 1. 严肃，不严厉 (Serious, Not Stern)
- **专业克制的视觉**：一种字体（Inter）+ 一种等宽（JetBrains Mono）+ 单色调
- **温和的语气**：所有微文案像同事在和你说话，而不是按钮 + 工程术语
- **零装饰 emoji**：状态用纯色圆点 / 文字 / 极简 SVG 图标；emoji 仅在 AI 回复正文中由模型自然产生

### 2. 一屏说完一件事 (One Job Per Screen)
- 每屏只承担**一个用户意图**：看公司 / 看一个员工 / 看一次运行 / 看复盘
- 任何屏幕的"主操作"按钮**永远只有一个**（参考 Linear 的 New Issue、Notion 的 New Page、Raycast 的 ⌘K）
- 每屏顶部 **永久全局命令面板** ⌘K，把"找/做/问"统一成一个入口（参考 Linear / Raycast / Vercel / Cursor）

### 3. 可预测的极简 (Predictable Simplicity)
- 所有列表、详情、画布的布局规律完全一致；学一个会全部
- 不发明新交互模式：用 Linear 已经教育用户的那套（侧栏 + 列表 + 详情滑出 + ⌘K）

### 4. 自然语言是一等公民 (NL Is a First-Class Citizen)
- **Captain 始终在屏幕右下角**（Notion AI / Cursor Composer / Raycast AI 风），随处可呼出
- 创建/编辑/调试/分析任何东西的旁边永远有 Captain 输入框（"用一句话告诉我"）
- 用户可以选择不用 — 但绝不强迫使用

### 5. 状态比按钮重要 (Status Over Buttons)
- 每个 Agent / 任务 / 运行的状态用 4 种颜色点表达（idle 灰 / running 蓝 / waiting 琥珀 / done 绿 / failed 红）
- 用 Linear 的 issue 状态徽章范式
- 让"系统在忙什么"永远一眼可见

### 6. 一切可撤销 (Everything Reversible)
- 任何写动作（发邮件、调 API、改 CRM、删除）有 **30 秒 Undo Toast**（Gmail 风）
- 任何 Agent 的过去 N 步都能被"回到那一步"再走一遍（LangGraph time travel）
- ⌘Z 是顶级公民

### 7. 解释胜过文档 (Explain, Don't Document)
- 不写帮助页；让 Captain 在用户卡住时主动说"看起来你想……，要我帮你做吗？"
- 错误不再是 Stack Trace，而是 "**这一步失败了，可能是因为…  | 让 Captain 修 | 显示原始错误**"

## Part C — 五大核心界面 v3（重做 v2 的卡通版）

### C-1. 工作区（Workspace）— 取代 v2 的"公司画布"

**定位**：用户登录后第一屏。**Linear 的 Inbox 风**，左侧栏 + 主列表 + 滑出详情，**不再是俏皮的环形画布**。

```
┌──────────────────────────────────────────────────────────────┐
│ ⌘K Captain  │  你的公司                          [➕ 新员工]  │
├─────────────┼────────────────────────────────────────────────┤
│ ▾ 员工      │  ● Sarah · 销售助理         5 分钟前 完成 1 任务 │
│   Sarah     │  ● Mark · 数据分析师        进行中 — 抓取季度数据│
│   Mark      │  ● Lily · 客服              空闲                  │
│   Lily      │  ● Jin · 招聘助理           等待审批 — 发送 offer │
│             │                                                  │
│ ▾ 任务      │  ─────────────────────────────────────────────  │
│   今日 12   │  本周完成  124   ↑ 18%                           │
│   本周 78   │  本月成本  $342   ▰▰▰▱▱▱▱▱▱▱  $1000 预算          │
│             │                                                  │
│ ▾ 设置      │                                                  │
└─────────────┴────────────────────────────────────────────────┘
```

**关键设计**：
- 状态用纯色圆点（4 像素），不用图标也不用 emoji
- 列表行高 36px，字号 14px，紧凑（Linear 标配）
- 选中行右侧滑出详情面板（Notion 风），不跳页
- 顶栏永久 **⌘K Captain 命令面板**，键盘党 1 秒到任意位置

### C-2. 员工详情（Member Detail）— 取代 v2 的"工作台三栏"

**定位**：点任意员工进入。**Notion 文档式排版**，所有"配置"以可读段落呈现，不是表单。

```
─────────────────────────────────────────────────
 S  Sarah · 销售助理
    负责对接潜在客户、整理需求、生成报价单。
    向 你 汇报 · 月预算 $200 · 本月已用 $43
─────────────────────────────────────────────────

  概览         任务         流程         记忆         设置

  ──── 进行中的任务 ───────────────────────────────
  ● 整理本周新增线索       3 分钟前  · 查看运行 →
  ● 给老客户发问候邮件     等待你审批 · 立即处理 →

  ──── 上周完成 ──────────────────────────────────
  ✓ 季度报价单            +12 份  · 详情
  ✓ 跟进未回复客户         34 通  · 详情
─────────────────────────────────────────────────
```

**关键设计**：
- 全部信息都是"可读句子"，不是 Form 字段
- "标签页"用 Linear 的下划线导航，不用 Tab 卡片
- 任何字段 hover 出现一个小铅笔；点击就地编辑或召唤 Captain

### C-3. 流程（Procedure）— 取代 v2 的"三色块 SOP"

**定位**：教员工"怎么做事"。**自然语言文档 + 行内动作 chip**，不再是 Scratch 积木块。

```
─────────────────────────────────────────────────
 Sarah 的工作流程                              [▷ 试运行]

 当 [📨 收到包含"报价"的邮件] 时：
   1. [📖 读取] 邮件正文，提取产品 SKU 和数量
   2. [🔎 在 知识库] 中查找当前价目表
   3. 如果 客户是 [👑 VIP]：
        – [💰 应用] 9 折规则
   4. [✍️ 起草] 报价邮件
   5. ⏸ 等待 [👤 你] 审批
   6. [📤 发送] 邮件给客户
   7. [📝 在 CRM] 中创建 "报价已发送" 任务

 [+ 添加一步]    或者：[💬 让 Captain 写一段]
─────────────────────────────────────────────────
```

**关键设计**：
- 主体是**自然语言段落 + 行内动作 chip**（chip = 灰底圆角 4px、SVG 图标 + 文字）
- 拖动一步可调顺序；hover 出现 ✕ 删除
- 复杂分支才会折叠成可展开节点
- "高级模式" 切到 LangGraph 节点画布（仅给开发者）

> 这一版与 Notion DataBase + Linear Workflow + Vercel 设置面板同源，零卡通、零 emoji 装饰。

### C-4. 运行视图（Run）— 取代 v2 的"思考剧场"

**定位**：实时看 Agent 干活；一段时间后变成可分享的回放。**左侧 Chat + 右侧 Workspace** 经典双栏，整体风格如 **Cursor + Linear**。

```
┌──────────────────────────────┬───────────────────────────────┐
│  Sarah · 整理本周新增线索      │ 工作区                         │
│  开始 14:23 · 用时 1m 47s     │ ─────────────────────────────  │
│                              │ [浏览器] [终端] [文件] [diff]  │
│  Captain: 我已经查到了 38 条  │                               │
│  线索，正在按行业分类。        │ ┌─ live preview ─────────┐    │
│                              │ │ (current page snapshot) │    │
│  📊 调用 sheets.read          │ │                         │    │
│  ✓ 成功                       │ └─────────────────────────┘    │
│                              │                               │
│  ⏸ 我想给 5 个高优线索发邮件   │ 时间轴                         │
│  [批准 5 封]   [我先看看]      │ ●─●─●─●─●─◐─○─○                │
└──────────────────────────────┴───────────────────────────────┘
```

**关键设计**：
- 左侧的 Chat 流参考 **Cursor Composer / Claude / Gemini** 的"工具气泡"
- 审批卡片**不弹 Modal**——直接在对话流中嵌入两个按钮
- 右侧 4 个 Tab 永远存在，不要在没数据时隐藏（避免位移）
- 顶部一行 metadata（用时 / 模型 / 成本）始终在场

### C-5. 复盘（Replay）— 取代 v2 的"60 秒视频回放"

**定位**：跑完之后；可分享只读链接。**Linear Issue 详情 + 时间轴**风。

```
─────────────────────────────────────────────────

 用时 1m 47s · 调用 12 次 · 花费 $0.041
 [复制可分享链接 ↗]   [比较其他运行]   [回滚 ↺]
─────────────────────────────────────────────────

 时间轴
 ●─●─●─●─●─●─●─●─●─●─●─●     拖动指针看任意时刻

 ╭─────────────────────────────────────────────╮
 │ 14:23:01  📨 收到任务                         │
 │ 14:23:02  🧠 计划：1) 抓取 Sheet 2) 分类 3)…  │
 │ 14:23:04  📊 sheets.read("Q2 leads")  +12ms  │
 │ 14:23:18  📊 crm.search("active=true")  +73ms │
 │ ...                                          │
 │ 14:24:48  ✓ 任务完成，已写回 38 条记录到 CRM  │
 ╰─────────────────────────────────────────────╯
```

**关键设计**：
- 时间轴在顶部，hover 即看快照（参考 Stripe Dashboard 时间序列）
- 步骤卡片用 **Linear 的活动流**样式
- "复制可分享链接" 是核心按钮（差异化卖点：让老板 1 分钟看完一次工作）

## Part D — 全局命令面板 (⌘K)：v3 的灵魂

> 这是 v3 与 v2 最重要的差异之一。你应该把整个产品的入口收敛到 ⌘K，对标 **Raycast / Linear / Cursor / Vercel**。

```
⌘K  搜索员工、任务、命令，或问 Captain 一句话…

  最近               
   • Sarah 的本周报告  
   • 整理新增线索 (运行)  

  做点事              
   ➜ 雇一个员工          ⌘N
   ➜ 派一个新任务         ⌘T
   ➜ 看本月成本           ⌘B
   ➜ 切换公司             ⌘O
   ➜ 安装一个 MCP 工具    ⌘I

  问 Captain          
   ✦ "把上周客户反馈整理成周报"
   ✦ "帮我招一个负责小红书的员工"
   ✦ "Sarah 上周发的 5 封邮件分别是给谁的？"
```

3 个标签页（最近 / 做点事 / 问 Captain）—— 用户**永远不需要记住菜单结构**。

## Part G — 用户测试与可用性指标 v3

### 测试人群（替换 v2 的 10 岁孩子）
1. **3 名非技术管理者**（创业者 / 部门主管 / 小老板）
2. **3 名设计师 / 运营**（高审美但不写代码的人）
3. **3 名熟练开发者**（看产品快速上手能力，反向验证"高级模式"）

### 硬指标（OKR 写进里程碑）

| 指标 | 标准 | 测试方法 |
|---|---|---|
| **首次理解时间** | 用户进入产品 30 秒内能用一句话描述"这是干嘛的" | 录屏复盘 |
| **首次成功** | 非技术管理者从注册到看到首个真实结果 ≤ **8 分钟** | 录屏 + 计时 |
| **第 1 天留存** | ≥ **60%** | 埋点 |
