# §4 产品与技术方案

> 本章对应 `16_tech_benchmark.py` 与 `04_slo_latency_budget.py`、`05_alerts_capacity_erlangc.py`、
> `06_dedup_suppression_sim.py`，所有性能基准、推理延迟、单路成本均可复现。

## Key Takeaways

1. **核心能力**：7×24 实时多模态监控——主播离开 60 秒内自动告警，**P50=42 秒、P90=58 秒、P99=72 秒**，可用性 SLA 99.9%。
2. **三层架构**：边缘采集（轻量推理 / 隐私脱敏）→ 云端融合（重模型 / 决策状态机）→ 多通道告警（短信/电话/微信/钉钉/飞书/企微/Webhook）。
3. **七模块算法栈**：人脸检测、人形检测、Re-ID、活体检测、行为识别、声纹/VAD、决策融合，**端到端在岗判定 F1 = 99.25%**，误报率 0.6%、漏报率 0.4%。
4. **反作弊为差异化核心**：通过 OSNet Re-ID + 活体 + 声纹三重校验解决"照片回放/数字人替播/换人挂播"，**反作弊均分 94（vs 通用 CV 38 / 规则脚本 21）**。
5. **单位经济友好**：单路推理成本由 Y1 ¥260 降至 Y5 ¥70（年降 28%），直接对应 78% 稳态毛利率。

## §4.2 系统架构（Edge / Cloud / Notify 三层）

![Edge / Cloud / Notify 三层架构](liveguard/docs/images/liveguard_architecture.png)

> 设计要点：
> 1. **隐私优先**：边缘层只输出 128/256 维特征向量与元数据，原始视频不上云（可私有化留存本地），符合 PIPL "最小必要" [S-010]。
> 2. **断网容灾**：边缘层独立完成 P0 告警（无需云端），云端用于复核与统计。
> 3. **横向扩展**：Cloud 层基于 Kubernetes + Triton Inference Server，单 GPU 节点（T4/A10）可服务 ≈ 80–120 路 1080p 直播。

### §4.3.1 决策融合状态机

| 输入信号 | 权重 | 阈值 |
|---------|------|------|
| 人脸检测置信度 | 0.30 | conf ≥ 0.55 |
| 人形检测置信度 | 0.20 | conf ≥ 0.50 |
| Re-ID 余弦相似 | 0.20 | sim ≥ 0.70 |
| 活体检测得分 | 0.10 | score ≥ 0.85 |
| 行为识别 | 0.10 | 综合标签 |
| 声纹 / VAD | 0.10 | 检测到目标说话人 |

复合分数 ≥ 0.65 → 在岗；< 0.35 → 离开；中间区间进入 5 秒缓冲。状态机在"离开"持续 60 秒后触发首次告警，120 秒升级，300 秒电话叫醒。

## §4.6 护城河（技术侧）

1. **垂类数据闭环**：Y2 末 50 万小时直播视频标注库（多场景、多镜位、多光照、多语言），新进入者需 18+ 个月追赶。
2. **算法 IP**：直播垂类反作弊算法 ≥ 8 项发明专利 + 5 项软著。
3. **平台生态对接**：抖音电商服务商认证、淘宝灯塔/千牛接入、快手磁力金牛接入。
4. **客户切换成本**：告警流程嵌入 MCN 排班、历史告警数据资产化（KPI/审计）。
5. **合规备案**：算法备案、PIPL 第三方审计、ISO 27001、SOC 2 Type II。
