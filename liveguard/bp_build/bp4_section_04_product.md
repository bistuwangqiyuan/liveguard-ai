# §4 产品与技术方案

> 本章对应 `16_tech_benchmark.py`、`04_slo_latency_budget.py`、`05_alerts_capacity_erlangc.py`、
> `06_dedup_suppression_sim.py`，所有性能基准、推理延迟、单路成本均可复现。

## Key Takeaways

1. **核心能力**：7×24 实时多模态监控——离开 60 秒内告警，**P50=42s / P90=58s / P99=72s**，可用性 99.9%。
2. **三层架构**：边缘采集（隐私脱敏）→ 云端融合（决策状态机）→ 多通道告警；**原始视频不出端**。
3. **七模块算法栈**：人脸/人形/Re-ID/活体/行为/声纹/决策融合，**端到端在岗判定 F1 = 99.25%**，FAR 0.6% / FNR 0.4%。
4. **反作弊为差异化核心**：Re-ID + 活体 + 声纹三重校验破解照片/回放/数字人/换人，反作弊均分 **94**（通用 CV 38 / 规则脚本 21）。
5. **产品平台化**：在监控之上扩展风控 OS、可信认证 API、RegTech，统一在一个技术底座（§1 四层）。

## §4.2 系统架构（Edge / Cloud / Notify 三层）

![Edge / Cloud / Notify 三层架构](liveguard/docs/images/liveguard_architecture.png)

> 设计要点：① **隐私优先**——边缘层只输出 128/256 维特征向量与元数据，原始视频不上云（符合 PIPL 最小必要）；② **断网容灾**——边缘独立完成 P0 告警；③ **横向扩展**——K8s + Triton，单 GPU 节点服务 ≈ 80–120 路 1080p。

## §4.3 算法栈与模型选型

| 模块 | 模型选型 | 论文/开源 | 关键 KPI | 单帧延迟 |
|------|---------|-----------|----------|----------|
| 人脸检测 | RetinaFace + SCRFD | [S-030] | P=98.2% R=97.5% | 12 ms |
| 人形检测 | YOLOv8s | [S-031] | mAP@0.5=0.95 | 8 ms |
| 行人 Re-ID | OSNet(微调) | [S-032] | Rank-1=94.2% | 15 ms |
| 行为识别 | SlowFast R50 | [S-033] | Top-1=78.3% | 42 ms |
| 行为识别(SOTA) | VideoMAE-B | [S-034] | Top-1=80.7% | 55 ms |
| 活体检测 | Silent-Face-AS | [S-036] | ACER=2.1% | 10 ms |
| 声纹验证 | ECAPA-TDNN | [S-035] | EER=0.87% | 18 ms |

![算法栈延迟与端到端 KPI](liveguard/docs/images/charts/fig_16_tech_kpi.png)

**决策融合状态机**：人脸(0.30)+人形(0.20)+Re-ID(0.20)+活体(0.10)+行为(0.10)+声纹(0.10)；复合分 ≥0.65 在岗、<0.35 离开；离开持续 60s 首告警，120s 升级，300s 电话叫醒。
