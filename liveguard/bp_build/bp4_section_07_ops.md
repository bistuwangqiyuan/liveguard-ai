# §7 运营与组织计划

> 端到端运营体系：客户成功、技术运维（SRE/NOC）、数据/算法运营、供应链、合规运营。
> NOC 容量与告警抑制策略由 `05_alerts_capacity_erlangc.py` / `06_dedup_suppression_sim.py` 量化。

## Key Takeaways

1. **NOC 7×24**：Erlang-C 测算——10,000 路并发仅需 **36 名坐席**达成 90% 告警 60 秒内响应；坐席需求亚线性增长。
2. **告警去重/抑制**：推荐 W=300s 自适应策略，平衡降载与漏报；P0 告警不进抑制窗口直通。
3. **三层客户成功**：Onboarding → Adoption → Renewal & Expansion；目标 NRR ≥ 145%、NPS ≥ 70。
4. **边缘硬件供应链**：4 家 ODM 战略合作（地平线/瑞芯微/寒武纪/华为昇腾），月产能弹性 500–5,000 台。

## §7.3 数据与算法运营

```
客户授权 → 边缘端难例采样（仅特征向量）→ 匿名样本池
       → 主动学习 + 人工标注（100% 质检）→ 模型微调
       → A/B → 灰度 → 全量 OTA
```

- 数据增长：Y1 5 万 / Y3 50 万 / Y5 500 万小时（沉没资产 5–7.5 亿元，§3 护城河）。
- 标注团队：30 人专属 + 200 人外包；核心模型周更，专项月更，5 分钟一键回滚。

## §7.5 合规与安全运营

- 季度法律风险评审 + 月度数据治理审计 + 年度 ISO 27001/SOC 2 续审；政策预警 24 小时同步。
- 三防一体（WAF/DDoS + 零信任 + 加密/DLP）；季度红队渗透；年度等保三级测评。
- 数据生命周期：采集（告知+同意+最小化）→ 传输（TLS 1.3 + 国密）→ 存储（分级加密+灾备）→ 销毁（离网 30 天内）。
