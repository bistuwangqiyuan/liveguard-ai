# §10 风险分析与对策（含合规与伦理）

> 本章基于全方位风险扫描（Macro / Market / Tech / Operational / Legal / Financial / ESG），
> 配合 §8.5 敏感性与 §9.4.3 蒙特卡洛，给出 **风险矩阵 + 应对预案 + 触发指标 + 责任人**。

## Key Takeaways

1. **24 项核心风险全量盘点**，按"概率 × 影响"分级——Top 3 红色风险均有 ≥3 套预案。
2. **合规护城河**：ISO 27001 / SOC 2 Type II / 等保三级 / 算法备案 / 数据出境评估资质。
3. **隐私优先架构**：边缘算法只输出"在/不在 + 时间戳"特征向量，不上传原始视频；可 100% 本地化部署。
4. **三道防线治理**：业务层风控、合规与法务、内审与外部董事审计委员会。

## §10.3 三大红色风险深度预案

### R01 / R02 — 监管与合规
情景：网信办出台《直播安全监控类 SaaS 管理细则》。三层预案——
- **技术**：默认本地化推理，上传仅 < 1KB/分钟 元数据；私有化部署版（K8s/Docker Compose）。
- **合规**：提前 6 个月备案；完成等保三级 / ISO 27001 / SOC 2；主动接入网信办内容安全平台。
- **业务**：监管驱动反成护城河——免费/低价竞品因合规成本上升退出；参与行业标准制定。

### R03 — SLA 违约（漏报误报）
数据飞轮（匿名样本回流 → 月度迭代）+ 难例挖掘 Pipeline（置信度 0.4–0.7 样本人工标注）+ SLA 违约信用补偿 + NOC 30 分钟介入。

### R04 / R06 — 平台/巨头竞争
嵌入而非对抗（成为平台官方合作伙伴，提供 SDK）+ 跨平台中立（覆盖全平台）+ 垂直深耕（教培/医疗/政务平台不涉足）。

## §10.4 合规与伦理（Compliance & Ethics）

### 中国法规清单
PIPL、数据安全法、网络安全法、生成式 AI 服务管理暂行办法、算法推荐管理规定、数据出境安全评估办法、互联网直播服务管理规定、GB/T 35273。

### 海外法规对照
GDPR (EU)、CCPA/CPRA (CA)、Singapore PDPA、EU AI Act（视频监控 → 高风险 AI，短期不进入欧盟）。

### 伦理框架（《AI 伦理白皮书 v1.0》六原则）
1. **透明**：所有识别动作有可解释报告
2. **公平**：定期算法偏见审计并公开
3. **隐私**：默认本地化、最小化、特征不可逆
4. **安全**：等保/ISO/SOC 三认证 + 季度红队
5. **可控**：人类始终在回路（HITL），客户可一键关闭
6. **问责**：AI 伦理委员会 + 年度透明度报告

### 治理结构（Three Lines of Defense）
第一道：业务部门（嵌入风控）；第二道：合规与法务（CCO/CLO/DPO）；第三道：内审 + 外部审计（普华永道/安永）。
