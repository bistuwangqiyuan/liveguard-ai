# §8 创立所需资源与里程碑

> 本章取代传统"限定投入资金 + 创始人时间"的评估模式，改为**自底向上列出把公司做出来需要的资源**
> （团队 / 算力 / 数据标注 / 资质牌照 / 平台合作 / 资本），再**由资源需求倒推天使轮规模**。
> 全部来自 `17_resource_requirements.py`。

## Key Takeaways

1. **天使轮规模是"算出来的"**：天使覆盖期（精益 9 个月）资源 ¥908 万 + 15% 缓冲 = **需求 ¥1,044 万 → 天使轮设定 ¥1,000 万**（一致）。
2. **5 年累计资源**：人力 ¥7.26 亿 + 算力 ¥0.80 亿 + 标注 ¥5.51 亿 + 资质 ¥218 万 + 平台 BD ¥600 万，由 5 轮融资 ¥20.6 亿覆盖且全程现金为正。
3. **数据飞轮是最大沉没资产**：5 年累计 500 万小时标注（¥5.5 亿），构成核心护城河。
4. **资质牌照清单**：算法备案、等保三级、ISO 27001、SOC 2 Type II、增值电信、PIPL 审计——合计 ¥218 万，是合规护城河的门票。
5. **v4.0：个人资本 ≠ 公司资本**（见下表）——公司天使需求 ¥1,000 万独立于王启源 Pre-Angel ¥500 万。

### 个人 vs 公司资本（v4.0）

| 口径 | 金额 | 用途 |
|------|-----:|------|
| **王启源 Pre-Angel** | ¥500 万 | 个人股权投入（Post ¥2,500 万）|
| **王启源时间机会成本** | 5×¥100 万/年 | 个人经济 bankroll（非公司 P&L）|
| **公司天使轮（外部）** | ¥1,000 万 | §8.1 资源倒推，覆盖 9 个月精益跑道 |
| **凯利建议总承诺** | **≈ ¥68 万** | 1/4 Kelly，见 §11b（非 1,000 万全投入）|

## §8.1 天使覆盖期资源 → 倒推天使轮（来源 `17`）

> 天使轮的使命：用精益资源跑到 Seed 里程碑（MVP→GA、算法备案、ARR 500 万、1 家平台 PoC）。

| 资源项 | 金额 | 说明 |
|--------|-----:|------|
| 核心团队（12 人 × 9 月）| ¥405 万 | 创始 7 人 + 首批工程/标注 5 人 |
| 数据采集与标注（2.5 万小时）| ¥275 万 | MVP 冷启动语料 |
| 算力（6 GPU 节点 × 9 月）| ¥97 万 | 训练 + 早期推理 |
| 首程资质（算法备案 + 增值电信）| ¥50 万 | 进入大客户销售的门票 |
| 办公 / 法务 / 杂项（9 月）| ¥81 万 | — |
| **小计** | **¥908 万** | — |
| + 风险缓冲 15% | +¥136 万 | — |
| **= 天使轮需求** | **¥1,044 万** | → **天使轮设定 ¥1,000 万** ✓ |

![天使覆盖期资源与 5 年资金充足性](liveguard/docs/images/charts/fig_17_resources.png)

> 这就是"天使 ¥1,000 万"的来源——不是拍脑袋的投入上限，而是把首程资源逐项加总倒推得到。
