# §5 商业模式与定价策略

> 数据基础：`08_pricing_model.py`、`02_unit_economics.py`、`03_roi_merchant.py`、`09_cohort_retention.py`。

## Key Takeaways

1. **四象限收入模型**：①SaaS 三层订阅（Y1 88% → Y5 68%）、②按量付费/加路（5%→14%）、③平台分润/API（3%→13%）、④硬件/私有化（4%→5%）。
2. **定价中位**：Starter ¥199/月、Pro ¥599/月、Enterprise ¥250,000/年；客户结构 70/25/5 → **加权年 ARPU = ¥15,969**。
3. **单位经济极佳**：**LTV/CAC：Starter 11.1× / Pro 22.0× / Enterprise 112.8×**，混合回本期 < 6 个月，远超 SaaS 健康基准（≥3×）。
4. **客户 ROI 显著**：个人主播 ROI = 75×、回本 4.8 天；中型 MCN ROI = 146×、回本 2.5 天；头部 MCN ROI = 26×、回本 13.7 天。
5. **留存稳健**：金额口径 GRR = 85%，NRR 由 Y1 95% 提升至 Y5 129%。

## §5.1 四象限收入模型

```
                ┌──────────────────────┬───────────────────────┐
                │ 一次性 (One-time)    │ 持续性 (Recurring)     │
直接卖产品/服务 │ ④ 硬件/私有化 ~5%    │ ① SaaS 订阅 ~70%       │
间接 (Indirect) │ Implementation Fee   │ ③ 平台分润/API ~15%    │
                ├──────────────────────┴───────────────────────┤
                │ ② 按量付费：加路/加保留期/API 调用 ~10%        │
                └──────────────────────────────────────────────┘
```

## §5.2 三层 SaaS 定价

> 来源：`08_pricing_model.py`。

| 档位 | 月单价 | 年单价 | 客户结构 | 月流失 | 价格弹性 e | 目标客户 |
|------|-------:|-------:|--------:|------:|----------:|---------|
| **Starter** | ¥199 | ¥2,388 | 70% | 5.0% | 1.5 | 个人主播 / 小商家 |
| **Pro** | ¥599 | ¥7,188 | 25% | 2.5% | 0.8 | 中型 MCN / 品牌自播 |
| **Enterprise** | ¥20,833 | ¥250,000 | 5% | 0.8% | 0.4 | 头部 MCN / 平台 |

```
加权年 ARPU = 70%×2,388 + 25%×7,188 + 5%×250,000
            = 1,672 + 1,797 + 12,500 = ¥15,969 / 年
```

![定价与价格弹性](liveguard/docs/images/charts/fig_08_pricing.png)

> 注：5% Enterprise 客户贡献 78% 的加权 ARPU——商业模式呈"长尾驱动"。

## §5.3 价格弹性与折扣策略

| 档位 | 弹性 e | 含义 |
|------|-------:|------|
| Starter | 1.5 | 高弹性，降价驱动量增但贡献毛利下降 |
| Pro | 0.8 | 中等弹性，降价 10% 需求 +8% |
| Enterprise | 0.4 | 低弹性，价值/合规驱动，价格非主因 |

**折扣策略**：Starter 年付 8 折引导预付；Pro 年付 8 折 + 14 天试用 + 推荐返佣；Enterprise 阶梯返点（50/100/300 万 → 95/90/85 折）。

## §5.5 客户 ROI 计算器（演示用）

> 来源：`03_roi_merchant.py`。

```
现有损失 = 年 GMV × (空播损失率 9.0% + 罚款率 2.5%)
部署后损失 = 年 GMV × (9.0%×30% + 2.5%×16%) = 年 GMV × 3.1%
年节省 = 现有损失 − 部署后损失；ROI = (年节省 − 工具年费) / 工具年费
```

| 客户分层 | 日均 GMV | 年节省 | 工具年费 | ROI | 回本天数 |
|---------|---------:|-------:|---------:|-----:|---------:|
| 个人主播 / 小商家 | ¥6,000 | ¥18.1 万 | ¥2,388 | **75×** | 4.8 天 |
| 中型 MCN / 品牌自播 | ¥35,000 | ¥105.8 万 | ¥7,188 | **146×** | 2.5 天 |
| 头部 MCN / 大型电商 | ¥220,000 | ¥665.3 万 | ¥250,000 | **26×** | 13.7 天 |

![客户 ROI 与敏感性](liveguard/docs/images/charts/fig_03_roi_customer.png)

> 销售可现场用客户自己的 GMV 演示——任意客群最长回本不超过 14 天。敏感性测算：即使客户当前空播损失率仅 1%，Pro 档 ROI 仍 ≥ 5×。

| 收入类型 | Y1 | Y3 | Y5 |
