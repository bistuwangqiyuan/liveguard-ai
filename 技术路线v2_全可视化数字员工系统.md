# 技术路线 v2 — 全可视化数字员工管理系统
> 子标题：**对 10 岁孩子也友好的、世界一流的、对话+画布双驱动的智能体可视化平台**
>
> 与 v1 的关系：v1 是面向工程师的 PoC（Paperclip+Hermes+assistant-ui+Langfuse），v2 是面向**产品/最终用户**的全可视化路线，调整 70% 决策、保留 30% 内核。
>
> 调研基线（2026-04-22 真实数据）：
> - **Coze Studio**（字节，Apache-2.0，20k+★）—— 全可视化 Agent 开发平台，与商业版 Coze 同源
> - **Letta ADE**（Apache-2.0）—— 三栏式 Agent IDE，记忆/上下文可视化
> - **Microsoft Copilot Studio** —— 企业级零代码 Agent 设计原则范本
> - **RAGFlow 0.24**（Apache-2.0）—— 已进入"Agentic 纪元"，含 ScenarioPlanner 自然语言生成画布
> - **Voiceflow** —— 设计师友好的对话流程画布
> - **Stack AI / Wordware** —— 自然语言→工作流的"Vibe Workflow"先行者
> - **v0 / Bolt / Lovable** —— prompt-to-app 的生成式 UI
> - **Manus / Genspark / Felo** —— 任务时间线型 Agent
> - **Scratch / MIT App Inventor / Blockly** —— 块式编程 UX 教科书
> - **Pydantic AI AG-UI** —— Generative UI 协议雏形

## Part A：路线 v1 复盘 — 哪些决策必须修正

| v1 决策 | 暴露的问题 | v2 修正 |
|---|---|---|
| 把 Paperclip 的 React UI "fork 改皮"作为控制台 | Paperclip 偏极客向，4 视图不足以承担"全可视化创建/编辑 Agent"，二开成本反超自研 | **只保留 Paperclip Server 作为后端核心服务**，前端 100% 自研；UI 设计语言重做 |
| assistant-ui + tool-ui 作为前端骨架 | ✅ 这条保留，但要加：**画布层（React Flow）+ 块式编辑（Blockly-style）+ 生成式 UI 协议（AG-UI 风）** | 升级为「**Chat × Canvas × Blocks × GenUI**」四位一体 |
| Hermes 作为单一运行时 | Hermes 是 TUI/消息接入向，**单实例假设难以撑住多租户**；其 `delegate_task` 还在 RFC 阶段 | 改为「**LangGraph 主 + Hermes 可选**」双轨：LangGraph 做主流多租户场景，Hermes 给"个人 AI 员工"场景 |
| 沙箱用本地 Docker（PoC）→ Daytona（V1） | Daytona 在国内不通；PoC→V1 跨度过大 | 改为「**E2B（云沙箱默认）+ 本地 Docker（私有化）+ 自研 Firecracker（V2 后期）**」三选一可切 |
| Langfuse 嵌 iframe | 内嵌体验差、品牌错位、无法做 10 岁孩子能看懂的"Agent 复盘" | 自研「**回放剧场**」组件（时间线 + 步骤卡片 + Token/Cost 浮层），底层仍写 OTel→Langfuse |
| 编排走 Flowise/Langflow 二开（备选） | Flowise/Langflow 节点过密、面向开发者 | 改为「**意图画布**」：节点数永远不超过 12 个，背后是 LangGraph |
| 没有"AI 副驾"贯穿 | 所有创建/调试都需用户主动操作 | 全程加 "**Captain**"（AI 副驾）：自然语言 → 自动生成 Agent / 自动连工具 / 自动写 SOP / 自动建数据库表 |
| 没考虑"模板市场" | 用户冷启动太难 | 必须做 **Template Marketplace**（员工模板、SOP 模板、行业方案） |
| 没考虑"语音/移动端" | 老板/家长不会一直坐电脑前 | 加移动端 PWA + 语音对接（Web Speech API + 可选 Vapi） |

## Part B：核心设计原则 —「10 岁孩子可用」的 7 条铁律

灵感融合：**Scratch 块式范式 + Apple HIG + Microsoft Copilot Studio 设计框架 + Linear 极简 + Notion 模板生态 + Manus 任务时间线 + Pydantic AG-UI Generative UI**。

### 原则 1 — 一张画布管全部 (One Canvas To Rule Them All)
- 用户不必在"应用 / Agent / 工作流 / 知识库 / 工具" 5 个独立菜单间跳转
- 一张「**公司画布**」放所有 Agent 头像 + 它们之间的箭头；点任意一个进入它的"工作台"
- 灵感来源：Paperclip Org Chart × Scratch 舞台 × Figma 多人协作

### 原则 2 — 万物可对话生成 (Everything Spawned by Chat)
- 所有"创建" 类按钮的旁边，永远有一个"**告诉我你想要什么**"输入框
- 输入「我要一个负责小红书运营的员工」 → 自动生成完整 Agent（含 SOP、工具、知识库）
- 输入「这个员工每天 9 点把昨天数据发我邮箱」 → 自动建定时任务 + 邮件工具
- 灵感：Coze Studio 自然语言生成 / RAGFlow ScenarioPlanner / v0.dev / Stack AI Ask Workflow

### 原则 3 — 三色块式 SOP (Tri-Color Blocks)
- Agent 的"标准操作流程（SOP）"用三种颜色块表达：🟢 触发（when） · 🔵 步骤（do） · 🟡 决策（if）
- 任意一块可双击展开为"对话编辑器"，让 Captain 帮你重写
- 灵感：Scratch / Blockly / Voiceflow Conversation Block

### 原则 4 — 角色是头像不是配置 (Personas, Not Configs)
- 不要让用户配置"system prompt / temperature / max_tokens / tools[]"
- 让用户选/捏一个"**头像**"——头像背后即包名包系统提示词、模型、工具、记忆、KPI
- 头像可"换装"：把"销售经理"头像拖到"客服"模板上 → 一键变成"销售客服" Agent
- 灵感：Letta ADE 三栏 + Genspark Agent 卡片 + Apple Memoji

### 原则 5 — 看得见的脑子 (Visible Brain)
- 任何 Agent 在工作时，右侧"**思考剧场**"实时播放：它在想什么 / 看了什么 / 用了什么工具 / 写了什么文件
- 关键时刻自动暂停，弹出 **审批卡**（绿/红两个大按钮 + "为什么"按钮）
- 完成后生成 **可分享的"工作录像"**（含可拖动时间轴 + 跳转任意步骤）
- 灵感：Manus 时间线 / OpenHands Workspace / Cline Approval / Letta ADE 上下文窗

### 原则 6 — 错了能撤销 (Undo Anything)
- 任何 Agent 的写操作（写文件、发邮件、调 API、修改 CRM）都进"**回收站**"
- 老板能 1 秒"回滚到 5 分钟前"
- 灵感：Notion 版本历史 / LangGraph time travel / Figma History

### 原则 7 — 别让用户读字 (Show, Don't Tell)
- 关键状态用 emoji + 颜色 + 微动画（pulse / spinner / confetti）
- 错误信息用"**Captain 帮你修**"按钮替代红色 Stack Trace
- 文档不超过 3 行，3 行内说不清的，给一个 30 秒视频
- 灵感：Linear 状态栏 / Apple HIG / Scratch 错误提示

> **可用性硬指标**：一个 10 岁孩子在不读任何文档的情况下，5 分钟内必须能完成「创建一个员工，让它每天告诉我天气」。

### D-3. SOP 编辑器（Tri-Color Blocks）
**作用**：教员工"做事步骤"的可视化编辑器；用块式编程范式。

```
┌──────────────────────────────────────────────────┐
│  [🟢 当用户在群里 @ 我]                            │
│      ↓                                            │
│  [🔵 读取消息内容]                                │
│      ↓                                            │
│  [🟡 判断是不是问题]                              │
│      ├── 是 → [🔵 在知识库搜索] → [🔵 回复答案]   │
│      └── 否 → [🔵 礼貌回复"收到"]                 │
│      ↓                                            │
│  [🔵 把对话存档]                                  │
└──────────────────────────────────────────────────┘
```

- 任何块支持「**双击改成自然语言**」：弹出对话框，Captain 帮你重写
- 任何块支持「**录屏式调试**」：选中→点 ▶️ 即在右侧 思考剧场 单步执行
- 块的颜色严格遵循 Scratch/Blockly：触发=绿、动作=蓝、判断=黄、停止=红

### D-4. 模板市场（Template Marketplace）
**作用**：让"招聘新员工"像"换主题"一样简单。

每个模板卡片包含：
- 头像 + 一句话职责（"小红书内容生成员工"）
- 30 秒动图演示
- ⭐ 评分 + 安装次数
- 「**直接雇佣**」按钮（一键复制到我的公司画布）

模板分 4 大类：
1. **个人助理**（行程、邮件、备忘）
2. **运营专员**（小红书、微信公众号、知乎、抖音）
3. **业务专家**（销售、客服、人力、财务、法务）
4. **行业方案包**（电商、SaaS、教培、医疗、工厂）

### D-5. 思考剧场 + 工作录像（Replay Theater）
**作用**：把 Agent 的执行变成"可拖动时间轴的视频"，老板/家长一目了然。

核心组件：
- **时间轴**：横向，每一步是一个圆点；带 mini 截图 hover 预览
- **步骤卡片**：左侧叠 Card；每张卡 = 一次思考/一次工具调用
- **三浮层**：Token / 成本 / 耗时
- **可分享**：生成只读链接，发给老板"看 1 分钟决定是否雇佣"
- **可对比**：两个 run 并排回放（Diff 模式，找回归 bug）

## Part E：技术栈最终决策（License 安全 + 可商业化）

### E-1. 必须避开的坑

| 项目 | 红线 | v2 替代 |
|---|---|---|
| **Dify** | "附加条款"禁多租户 SaaS 转售 + LOGO 限制 | ❌ 不再考虑作为基座；可对照 UX 范式 |
| **Flowise** | 自定义 LICENSE，商业 SaaS 限制 | ❌ 不作基座；可参考画布交互 |
| **AgentGPT** | GPL-3.0 污染 | ❌ |
| **Lobe Chat** | 自定义条款 + 历史商标限制 | ❌ 仅参考 UX |
| **n8n** | fair-code（禁商业 hosting 转售） | ⚠️ 仅"用户私有化时"作为可选触发器 |
| **AutoGen** | CC-BY-4.0 不是软件许可 | ⚠️ 仅作多 Agent 设计参考，不在生产路径 |
| **Open Agent Platform**（LangChain） | **2026 年已废弃** | ❌ 别选这条已死路线 |
| **Paperclip 自带前端** | 看似 MIT，但其品牌强绑定 + 极客向 UI 不利商业化 | ✅ 仅用其 Server，前端自研 |

### E-2. 推荐技术栈（最终版）

```yaml
前端基座:
  框架: Next.js 14 App Router (TypeScript, MIT)
  样式: Tailwind CSS + shadcn/ui (MIT)
  Chat 与 Tool 渲染: assistant-ui + tool-ui (MIT)
  画布: React Flow (MIT)
  块式编辑: Blockly (Apache-2.0) 或自研轻量版
  数据/状态: TanStack Query + Zustand (MIT)
  动画: Framer Motion (MIT) + Lottie
  生成式 UI 协议: 自研 + 参考 Pydantic AG-UI

控制平面后端:
  Paperclip Server (MIT) — 仅作为 Org/Task/Cost/Audit 数据层
  自建 IAM 层 — Lucia Auth (MIT) 或 Better Auth
  RBAC — Casbin (Apache-2.0)
  数据库 — Postgres + pgvector (PostgreSQL License) / Drizzle ORM (Apache-2.0)

Agent 运行时:
  主轨: LangGraph (MIT) — 多租户场景默认
  备轨: Hermes Agent (MIT) — 个人/家庭场景，强记忆
  协议: MCP (开放标准)
  
Captain 副驾:
  Vercel AI SDK (Apache-2.0) — 流式 + 工具调用
  Pydantic AI (MIT) — 服务端结构化输出
  
沙箱:
  云端默认: E2B (Apache-2.0)  
  私有化默认: Docker + Sysbox
  长期 V2.5: 自建 Firecracker
  浏览器自动化: browser-use (MIT) / Playwright (Apache-2.0)
  
可观测:
  自研「回放剧场」(自有版权)
  存储后端: Langfuse 自托管 (MIT 核心) — OTel 协议接入
  评测: Promptfoo (MIT) 或自研

部署:
  容器: Docker Compose (PoC) → K8s + Helm (V1) 
  CI/CD: GitHub Actions
  CDN/边缘: Cloudflare 或自有
  
集成生态:
  社区工具: 直接接 MCP 生态（GitHub/Linear/Notion/Slack/飞书/企微）
  自动化触发: 用户私有化部署时可用 n8n（fair-code，但用户自部署不算转售）
```

## Part G：杀手级差异化（让世界记住你）

| 差异点 | 别人怎么做 | 你怎么做 | 难度 |
|---|---|---|---|

| **整个系统由 Captain 驯化** | 别家 Studio 让你拖节点 | Captain 全程坐在右下角；你说话、它干活；只在它做不到时才弹画布 | 高 |
| **可分享的"工作录像"** | Langfuse 给开发者看 trace | 一键生成"老板友好"的 60 秒视频回放 | 中 |
| **撤销一切** | Cline 只能 git diff | 任何写操作进回收站，1 秒回滚到 X 分钟前 | 中 |
| **公司画布的"组织演化"** | 静态 Org Chart | 老板可以拖动重组；Captain 提示"这个员工 7 天没干活了，要不要让它退休？" | 中 |

## Part H：可用性测试方法（强制写进流程）

每个里程碑结束都跑下面 3 类测试，否则不得进入下一里程碑：

1. **10 岁用户测试**：邀请 3 名 10 岁孩子，在不引导的情况下完成预设任务；记录卡点、面部表情、放弃次数。
2. **老板视角测试**：邀请 3 名非技术管理者，给他们一句话需求（"我要个员工帮我整理本周客户反馈"），看从注册到看到结果用时几分钟。
3. **30 秒理解测试**：给完全没接触过的人看产品 30 秒，关掉，让他描述"这是干嘛的"；如果描述不出"AI 员工帮我干活"，说明设计失败。

> **可用性硬指标（写进 OKR）**：
> - 10 岁用户首雇佣员工时长 ≤ 5 min
> - 老板从注册到看到首个真实结果 ≤ 10 min
> - 30 秒理解测试通过率 ≥ 80%
> - 用户首日留存 ≥ 60%
